# 2.1 实验分组要求

5人左右一组,大家共同协商建立数据模型:ER图以及相关数据库,相关的实体类,系统整体架构、接口等。每个人可以在以下内容中选择1~n个子系统进行独立的设计,架构(注意,至少三层架构)并最终实现它们。

# 2.2实验内容

导引每小组合作利用AI工具(比如华为的code arts)快速完成以下在线学习系统(可以任意选择某科目,比如高数)的设计与实现,最终要上机运行检查

[x] 1.题目管理子系统(题库子系统):完成教师管理,教师对题目的增删改查。要求包含至少三种题型:选择,填空,以及带贴图的主观题(回答也用作图形式),做好题型以后有可能要扩展的准备

[x] 2.题目性质管理子系统(进行每道题的答案管理,题目所涉及的知识点管理难易度管理,题目所在课程的章节管理,题目的出错率,易错点设置等等)

[x] 3.自动组卷子系统(自动组卷要能够设置总分,难易度以及选取题目种类和每种题数目之后,从题库中自动成卷,给出每题分值,题目包含所有知识点以及计算机学院实验指导书难易度匹配;

[x] 4.手动组卷子系统(手动组卷,出题时设置总分、难易度和题目类型,在在题库中手动选择每个题型的题目并设置分值,要求提交时自动检验所有的题本基目覆盖了大部分的知识点(85%以上)以及题目难度符合要求,成卷后自动计算总分)

[] 5.考试管理子系统(考场管理,考试发卷,限时考试,客观题自动判分,主观题教师给分,以及给总分,该部分选择和4互斤)

[] 6.学生管理子系统(学生的院系班级分配,学生以班级为单位选课,学生管理,学生情况分析(历次考试成绩波动图,错误知识点统计报告等))

[] 7.错题分析子系统(试卷分析(归纳错题知识点和难易度,给出全班每个知识点错误学生的占比分析报告,给出每个学生的错题分析,包括哪个章节以及知识点错误率情况等报告,并提醒学生在哪些部分应该加强,并附」正确答案

[] 8.学习巩固子系统(按照错题的数量,难易度,所涉及的知识点,随机给出n道相似知识点题目,并附上解析答案,给出学习效果)
